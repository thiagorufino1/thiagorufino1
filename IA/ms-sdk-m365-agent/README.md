# Microsoft 365 Agent for Teams

Agente em Python construído com o Microsoft 365 Agents SDK e hospedado com `aiohttp`. O projeto expõe um bot para Microsoft Teams e, no estado atual, usa Azure OpenAI para responder mensagens com streaming.

## Status atual

- Canal principal: Microsoft Teams
- Runtime: Python + `aiohttp`
- SDK: Microsoft 365 Agents SDK
- LLM atual: Azure OpenAI
- Resposta incremental: sim, via streaming
- Metadados de streaming configurados: feedback loop, `generated by AI` e sensitivity label

## Arquitetura

Fluxo atual:

`Teams -> bot Python -> Azure OpenAI -> resposta em streaming -> Teams`

Arquivos principais:

- [src/app.py](src/app.py): inicializa o servidor HTTP e publica `/api/messages`
- [src/agent.py](src/agent.py): handlers do agente, streaming e chamada ao modelo
- [src/config.py](src/config.py): leitura das variaveis de ambiente
- [src/prompts/chat/skprompt.txt](src/prompts/chat/skprompt.txt): prompt de sistema
- [src/prompts/chat/config.json](src/prompts/chat/config.json): parametros de completions
- [appPackage/manifest.json](appPackage/manifest.json): manifesto do app do Teams
- [infra/](infra): provisionamento Azure/Bot

## Pre-requisitos

- Python 3.10+
- VS Code com Microsoft 365 Agents Toolkit
- Conta Microsoft 365 para teste no Teams
- Recurso Azure OpenAI com um deployment disponivel

## Instalacao

```powershell
python -m venv venv
.\venv\Scripts\activate
pip install -r src/requirements.txt
```

## Configuracao

O toolkit gera e combina valores a partir dos arquivos em `env/` e do arquivo `.env` na raiz durante o fluxo de debug/deploy.

Variaveis principais usadas pelo bot:

- `AZURE_OPENAI_API_KEY`
- `AZURE_OPENAI_ENDPOINT`
- `AZURE_OPENAI_DEPLOYMENT_NAME`
- `AZURE_OPENAI_API_VERSION`
- `PORT`

Variaveis opcionais para metadados de streaming:

- `M365_AGENT_FEEDBACK_LOOP`
- `M365_AGENT_AI_LABEL`
- `M365_AGENT_SENSITIVITY_NAME`
- `M365_AGENT_SENSITIVITY_TYPE`
- `M365_AGENT_SENSITIVITY_SCHEMA_TYPE`

Exemplo:

```env
AZURE_OPENAI_API_KEY=***
AZURE_OPENAI_ENDPOINT=https://your-resource.openai.azure.com/
AZURE_OPENAI_DEPLOYMENT_NAME=gpt-4o-mini
AZURE_OPENAI_API_VERSION=2024-12-01-preview
M365_AGENT_FEEDBACK_LOOP=true
M365_AGENT_AI_LABEL=true
M365_AGENT_SENSITIVITY_NAME=Internal
M365_AGENT_SENSITIVITY_TYPE=https://schema.org/Message
M365_AGENT_SENSITIVITY_SCHEMA_TYPE=CreativeWork
```

## Como executar

### Microsoft 365 Agents Playground

No VS Code, use a configuracao:

- `Debug in Microsoft 365 Agents Playground`

Isso e bom para validar:

- recebimento de mensagens
- streaming de texto
- comportamento geral do handler

Limitacao importante:

O Playground nao e a melhor referencia visual para recursos como feedback loop, badge `generated by AI` e sensitivity label. Esses metadados sao emitidos pelo backend, mas a experiencia visual completa depende do cliente do Teams.

### Teams

No VS Code, use uma destas configuracoes:

- `Debug in Teams (Edge)`
- `Debug in Teams (Chrome)`
- `Debug in Teams (Desktop)`

Esse e o fluxo indicado para validar:

- instalacao do app no Teams
- streaming no cliente real
- renderizacao de metadados e recursos especificos de Teams

## Recursos implementados

- Handler de boas-vindas em `conversation_update`
- Handler de mensagens
- Streaming incremental de resposta
- Uso de `AsyncAzureOpenAI`
- Prompt e parametros separados do codigo
- Feedback loop configurado no stream
- Rotulo `generated by AI`
- Sensitivity label no stream final

## Observacoes sobre streaming no Teams

O projeto configura os recursos do SDK no objeto `context.streaming_response`. No cliente correto, isso permite:

- feedback loop para resposta gerada
- indicacao visual de conteudo gerado por IA
- envio de metadados de sensibilidade

Se isso nao aparecer no Playground, valide no Teams real antes de concluir que o backend falhou.

## Desenvolvimento futuro

Direcao mais provavel para este repositorio:

- substituir Azure OpenAI por Databricks Genie para o caso de uso "chat com dados Databricks"

Arquitetura planejada para esse cenario:

`Teams -> bot Python -> Databricks Genie Conversation API -> Teams`

Nesse desenho:

- o bot deixa de usar Azure OpenAI como motor principal de resposta
- o `conversation_id` do Genie precisa ser persistido por conversa
- o foco do agente passa a ser consulta e conversa sobre dados do Databricks
